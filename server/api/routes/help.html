School Dashboard Web API Help Page

Overview:

The CodeWave School API allows interaction with the platform's users, courses, notifications, and news. It provides a structured way to manage resources through CRUD operations.

Users can register, retrieve, update, and delete information. Courses can be created, modified, and removed. Notifications ensure real-time updates, and news keeps users informed.

Each route follows RESTful principles, ensuring clear and predictable interactions. Authentication and validation mechanisms should be implemented where necessary to maintain data integrity and security.

Key Features

  • User Management: Create, update, delete, and retrieve user profiles.
  • Course Management: Manage course listings, including details such as instructors, descriptions, and schedules.
  • Notification System: Deliver real-time or scheduled notifications to users, track read status, and manage alert types.
  • News Updates: Provide a way to post and manage news articles related to school events, announcements, or general updates.

Base URL

http://localhost:5000/

Request and Response Format

  • All request payloads must be sent in JSON format.
  • Responses are also returned in JSON format.

Common HTTP Status Codes

  • 200 OK – The request was successful.
  • 201 Created – A new resource was successfully created.
  • 400 Bad Request – The request was malformed or contained invalid data.
  • 401 Unauthorized – Authentication credentials were missing or invalid.
  • 403 Forbidden – The user does not have permission to access the requested resource.
  • 404 Not Found – The requested resource does not exist.
  • 500 Internal Server Error – A server-side error occurred while processing the request.

Available Routes:

Users Routes:

  • CREATE /users/id - Creates a new user
  • GET /users - Retrieve all users
  • GET /users/id - Retrieve a user by id
  • PUT /users/id - Updates a user
  • DELETE /users/id - Deletes a user

Courses Routes:

  • CREATE /courses/id - Creates a new course
  • GET /courses - Retrieve all courses
  • GET /courses/id - Retrieve a course by id
  • PUT /courses/id - Updates a course
  • DELETE /courses/id - Deletes a course

Notifications Routes:

  • CREATE /notifications/id - Creates a new notification
  • GET /notifications - Retrieve all notifications
  • GET /notifications/id - Retrieve a notification by id
  • PUT /notifications/id - Updates a notification
  • DELETE /notifications/id - Deletes a notification

News Routes:

  • CREATE /news/id - Creates a new news
  • GET /news - Retrieve all news
  • GET /news/id - Retrieve a news by id
  • PUT /news/id - Updates a news
  • DELETE /news/id - Deletes a news
