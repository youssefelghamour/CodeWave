School Dashboard Web API Help Page

Overview:

The CodeWave School API allows interaction with the platform's users, courses, notifications, and news. It provides a structured way to manage resources through CRUD operations.

Users can register, retrieve, update, and delete information. Courses can be created, modified, and removed. Notifications ensure real-time updates, and news keeps users informed.

Each route follows RESTful principles, ensuring clear and predictable interactions. Authentication and validation mechanisms should be implemented where necessary to maintain data integrity and security.

Key Features

  • User Management: Create, update, delete, and retrieve user profiles.
  • Course Management: Manage course listings, including details such as instructors, descriptions, and schedules.
  • Notification System: Deliver real-time or scheduled notifications to users, track read status, and manage alert types.
  • News Updates: Provide a way to post and manage news articles related to school events, announcements, or general updates.

Base URL

http://localhost:5000/

Request and Response Format

  • All request payloads must be sent in JSON format.
  • Responses are also returned in JSON format.

Common HTTP Status Codes

  • 200 OK – The request was successful.
  • 201 Created – A new resource was successfully created.
  • 400 Bad Request – The request was malformed or contained invalid data.
  • 401 Unauthorized – Authentication credentials were missing or invalid.
  • 403 Forbidden – The user does not have permission to access the requested resource.
  • 404 Not Found – The requested resource does not exist.
  • 500 Internal Server Error – A server-side error occurred while processing the request.

Available Routes

Users Routes:

  • CREATE /users/{id} - Creates a new user
  • GET /users - Retrieve all users
  • GET /users/{id} - Retrieve a user by id
  • PUT /users/{id} - Updates a user
  • DELETE /users/{id} - Deletes a user

Courses Routes:

  • CREATE /courses/{id} - Creates a new course
  • GET /courses - Retrieve all courses
  • GET /courses/{id} - Retrieve a course by id
  • PUT /courses/{id} - Updates a course
  • DELETE /courses/{id} - Deletes a course

  • GET /notifications/{id} - Retrieve a notification by id

  • DELETE /notifications/{id} - Deletes a notification

News Routes:

  • DELETE /news/{id} - Deletes a news article

API Commands Examples

This section provides example API commands to interact with various endpoints, including users, courses, notifications, and news. Use these examples as templates for making requests to the CodeWave API.

Users:

GET all users:

curl http://localhost:5000/users

CREATE a new user:

curl -X POST http://localhost:5000/users -H "Content-Type: application/json" -d '{ "id": 3, "email": "user3@email.com", "password": "password3", "firstName": "user3", "lastName": "OldLastName", "studentId": "128 011", "cohort": "19", "courses": [ { "id": "1", "name": "JavaScript Basics", "credit": 60, "start_date": "2024-10-01", "end_date": "2024-12-01", "duration": "3 months", "status": "In Progress", "isSelected": false }, { "id": "2", "name": "Node.js Basics", "credit": 50, "start_date": "2024-11-01", "end_date": "2024-12-01", "duration": "1 month", "status": "In Progress", "isSelected": false }] }'

GET a user by id:

curl http://localhost:5000/users/3

UPDATE a user:

curl -X PUT http://localhost:5000/users/3 -H "Content-Type: application/json" -d '{ "lastName": "NewLastName" }'

DELETE a user:

curl -X DELETE http://localhost:5000/users/3

Courses:

GET all courses:

curl http://localhost:5000/courses

CREATE a new course:

curl -X POST http://localhost:5000/courses -H "Content-Type: application/json" -d '{ "id":"5","name":"ES6", "credit":60, "start_date":"2024-09-01", "end_date":"2024-12-01", "duration":"3 months", "status":"Completed"}'

GET a course by id:

curl http://localhost:5000/courses/5

UPDATE a course:

curl -X PUT http://localhost:5000/courses/5 -H "Content-Type: application/json" -d '{ "name": "Updated Course Name" }'

DELETE a course:

curl -X DELETE http://localhost:5000/courses/5

Notifications:

GET all notifications:

curl http://localhost:5000/notifications

CREATE a new notification:

curl -X POST http://localhost:5000/notifications -H "Content-Type: application/json" -d '{ "id":"5", "author": { "id": "5debd764a7c57c7839d722e9", "name": { "first": "user", "last": "lastname" }, "email": "user.name@email.com", "picture": "http://placehold.it/32x32", "age": 25 }, "context": { "guid": "2d8e40be-1c78-4de0-afc9-fcc147afd4d2", "isRead": true, "type": "urgent", "value": "Lorem ipsum dolor sit amet, consectetur adipiscing elit." } }'

GET a notification by id:

curl http://localhost:5000/notifications/5

UPDATE a notification:

curl -X PUT http://localhost:5000/notifications/5 -H "Content-Type: application/json" -d '{"context.isRead": false}'

DELETE a notification:

curl -X DELETE http://localhost:5000/notifications/5

DELETE a news article:
